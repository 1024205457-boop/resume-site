# Wiki Schema — 姜慧男求职知识库

## 目的
为个人求职网站提供结构化的公司分析和岗位匹配内容。LLM 负责所有写入和维护。

## 目录结构
- `raw/` — 原始资料（JD、行业报告、产品文档），只读不改
- `pages/` — LLM 生成的分析页面
- `index.md` — 所有页面的目录
- `log.md` — 操作记录
- `schema.md` — 本文件，定义规则

## 页面类型

### 公司分析页 `pages/{company}-analysis.md`
- 公司 AI 业务概览
- 目标岗位分析（JD 拆解）
- 岗位匹配度分析（简历 vs JD）
- "我能贡献什么"

### 岗位匹配页 `pages/{company}-fit.md`
- JD 关键词 → 对应经历映射
- 优势项（强匹配）
- 待补项（弱匹配或缺失）
- 面试准备要点

## Ingest 流程
1. 读取 `raw/{company}/` 下的原始资料
2. 读取简历数据（`site/js/data/resume.js`）
3. 生成/更新对应的 pages
4. 更新 `index.md`
5. 追加 `log.md`

## 写作规范
- 使用中文
- 客观分析，不浮夸
- 用数据和事实支撑匹配度判断
- 页面之间用 `[链接文字](文件名.md)` 互相引用
